4. Deep Insights — Cross-Cutting Patterns

Configuration Contradictions

Port Configuration Mismatch: App listens on port 8000 but Jenkins exposes container on port 5000.

Why it contradicts: Causes connection failures during deployment.

Impact: medium

Fix: Either change app port to 5000 or update Jenkins port mapping to 8000:8000.

Stress Projection — "What Breaks at Scale"

Current baseline: Single-threaded file I/O limits to ~10-20 concurrent requests with p95 latency of ~100ms.

At 2x load: File locking contention causes timeouts for ~30% of requests.

At 5x load: Race conditions cause data corruption, users see inconsistent task lists.

At 10x load: Complete system deadlock from file access conflicts.

At 100x load: File system exhaustion, application crashes.

Horizontal scaling readiness: impossible - file-based state prevents scaling beyond single instance.

Implicit Contracts

JSON file is always readable/writable: Code assumes file operations never fail.

Breaks when: Disk permissions, corruption, or space issues.

Impact: Application crashes with unhandled exceptions.

Suggested fix: Add try/catch blocks and graceful error handling.

Implicit Contracts

Task IDs are sequential integers: ID generation assumes no concurrent creation.

Breaks when: Concurrent creation or deletion causes gaps/conflicts.

Impact: ID conflicts and gaps cause confusion.

Suggested fix: Use UUIDs or proper ID generation with locking.

Implicit Contracts

Flask development server handles concurrency safely: Code assumes single-threaded execution.

Breaks when: Multiple simultaneous requests cause race conditions.

Impact: Data corruption from race conditions.

Suggested fix: Use proper concurrency controls or switch to production server.

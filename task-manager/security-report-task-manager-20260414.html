Medium Severity (1)

Docker: No .dockerignore
Medium
File: Dockerfile Ecosystem: Docker

Description: The project contains a Dockerfile but no .dockerignore file. This means the entire build context (all files in the project directory) will be sent to the Docker daemon, increasing image size and potentially exposing sensitive files.

Impact:

  • Increased Docker image size and build time
  • Risk of accidentally including sensitive files (e.g., .env, .git, private keys)
  • Slower deployments due to larger context transfers
